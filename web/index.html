📥 Monthly JSON Import (Upsert)

Upload a monthly JSON report. The system uses a full relational model — participants and programs are upserted (matched by email/name), enrollments enforce a unique (participant + program) pair, and status records are appended whenever the status, location, or region changes. This preserves full history for point-in-time reporting, and lets participants be grouped into Program/Location/Region "boxes" below.

Frequently Asked Questions

💭
Initial Thoughts
▼
Riley, aren't you a social worker? Why are you developing apps now? ▼
  • By building the app in-house, we can directly address the needs of programs and build the technologies that will assist us best.
Why are we using an app for monthly reports? ▼
  • Monthly reports have historically been written in MS Word.
  • The primary issue with the Word-based system is that it does not quantify any data, which forces coordinators to manually count data from every report to translate it into information required by funders.
  • This is a very time-consuming process.
  • This app is designed to automate that process.
Why are we entering this data here and not ARMS? ▼
  • ARMS is a great tool for entering notes and tracking interventions.
  • Presently, it does not function reliably for extracting all the data that is required by our funders.

Getting Started & Data Management
▼
Something isn't working / I've found a bug in the app, what do I do? ▼
  • Send an email to rhunt@jhsnl.ca describing the issue to have it resolved as soon as possible.
I have a great idea about how to improve the app, what should I do? ▼
  • Send an email to rhunt@jhsnl.ca that details the changes and/or additions that you'd like to see.
What is a JSON file? ▼
  • A JSON file (JavaScript Object Notation) is a lightweight format used to store structured data.
  • In this app, we use JSON files as a way for you to export, back up, or import your report data.
  • It acts as a digital "snapshot" of your entries that you can keep for your records or upload back into the app later if needed.
Where is my data stored? ▼
  • Your data is stored locally within the app's internal storage.
  • If you download an export, it will, by default, be saved in your computer's "Downloads" folder.
  • When you click "Submit," your report will be stored automatically in a protected folder on the JHSSHARE drive, accessible only by those who need to review the report.
📝
Entries
▼
Email Report is not working ▼
  • This function is designed to work when Outlook is installed and running on your PC.
  • If these conditions are not met, the email will not send.
  • If you are using the New Outlook, this function will draft an email for you automatically if the previously stated conditions are met.
  • However, the program will not automatically attach your report to the email.
What is the difference between "Add Entry" and "Archive Month"? ▼
  • Add Entry: This is for your daily data management; it appends new records to the list you are currently working on for the active month.
  • Archive Month: This is an end-of-month process that takes a snapshot of your entries, saves them to the archive, and allows you to start a fresh list for the next month.
How do I submit my report? ▼
  • Click the "Submit" button on the Entries page.
  • This will save a copy of the monthly report directly to a folder on the JHSSHARE drive that is accessible only by those who will need to review the report.
⚙️
Admin & Settings
▼
What is the Admin Page? ▼
  • The Admin page is used for reading the report files that are submitted and compiling them into larger quarterly and annual reports for funders.
  • This page is locked, not because it contains sensitive data, but because it holds no use for most program users.
How can I make the app easier to read? ▼
  • Navigate to Settings.
  • You can toggle Dark Mode for low-light environments and Color Blind Mode to swap red/green status colors for a blue/orange palette.
How do I set up default email recipients? ▼
  • In Settings, locate the "Default Recipient Email(s)" section.
  • Enter an address, click Add, and these addresses will be pre-filled whenever you use the "Email Report" function.
